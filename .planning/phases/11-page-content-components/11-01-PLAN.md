**FeatureCard.astro** -- Bento grid card for the Features section.
- TypeScript Props interface: `{ title: string; description: string; icon: string; colSpan?: number; rowSpan?: number }`
- Default colSpan=1, rowSpan=1
- Styling: bg-bg-secondary, border border-border, rounded-2xl, p-6
- Conditionally apply `md:col-span-2` when colSpan=2 and `md:row-span-2` when rowSpan=2 (only at md+ breakpoint to avoid mobile overflow -- see Pitfall 5 in research)
- Icon rendered as text/emoji at text-3xl mb-4
- Title: text-lg font-semibold text-text-primary mb-2
- Description: text-text-secondary text-sm leading-relaxed

**TerminalBlock.astro** -- Terminal-styled code block with copy-to-clipboard.
- TypeScript Props interface: `{ title: string; commands: string[] }`
- Join commands with newline for the data-code attribute: `const codeText = commands.join('\n')`
- Container: bg-bg-secondary, border border-border, rounded-xl, overflow-hidden
- Header bar: bg-bg-tertiary, border-b border-border, flex items-center justify-between px-4 py-2
  - Left: title in text-text-muted text-xs font-mono
  - Right: copy button with class "copy-btn", data-code={codeText}, aria-label="Copy {title} commands to clipboard"
  - Copy button text styling: text-text-muted hover:text-text-primary transition-colors text-xs font-mono
- Command lines: Each command in a flex row with $ prompt (text-text-muted select-none shrink-0) and command text (text-text-secondary)
- Code area: p-4 overflow-x-auto (handles long commands without page-level scroll)
- Font: font-mono text-sm throughout the code area
- Client-side script (Astro processed `<script>` tag, NOT is:inline):
  - querySelectorAll('.copy-btn') and addEventListener('click')
  - Use navigator.clipboard.writeText(code) in try/catch
  - On success: change button text to "Copied!", revert after 2000ms
  - On failure (no secure context): fallback to selecting text content via Range/Selection API
  - Astro auto-deduplicates this script even if TerminalBlock is used multiple times
