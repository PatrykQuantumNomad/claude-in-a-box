The SVG should depict this LEFT-TO-RIGHT data flow with 5 labeled nodes connected by arrows:
1. **Phone / claude.ai** (left) -- the user's device
2. **Anthropic API** (center-left) -- cloud relay layer
3. **Claude Code** (center) -- running inside the pod
4. **MCP Server** (center-right) -- kubernetes-mcp-server sidecar
5. **K8s Cluster** (right) -- the target cluster resources

Design guidance:
- Use rounded rectangles for each node (rx="8")
- Fill nodes with the bg-secondary color: use style="fill: oklch(0.20 0.01 260)" (or reference CSS var via style="fill: var(--color-bg-secondary)")
- Node borders: style="stroke: oklch(0.30 0.02 260)" matching border token
- Text labels: style="fill: oklch(0.95 0.01 260)" for primary text, smaller sublabels in secondary color
- Connecting arrows: style="stroke: oklch(0.70 0.15 250)" using the accent color, with arrowhead markers
- Define an `<defs>` block with an arrowhead `<marker>` element
- Keep the SVG clean and simple -- boxes with text and arrow lines between them
- Below the main flow, add a subtle row of 3 smaller items: "ServiceAccount", "NetworkPolicy", "PersistentVolume" -- shown as smaller pill-shaped elements below the main diagram to indicate supporting infrastructure
- DO NOT use fixed width/height attributes on the SVG element (let viewBox + w-full handle sizing)
- The diagram should be legible on mobile -- text should be at least 12px equivalent in the viewBox coordinate space
